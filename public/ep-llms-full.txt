## What EP Does
EP is the pre-execution trust layer for autonomous agents.
Before any action executes, EP validates it against a defined policy set
and returns a cryptographic proof_hash anchored on Base.
Boolean pass/fail in milliseconds. Framework agnostic. Asset agnostic.

## Five Pillars (ALL LIVE)
1. Identity — ERC-8004 on-chain agent registration on Base
2. Access — Policy sets enforced via POST /ep/validate
3. Management — Cryptographic proof_hash per action on Base
4. Reputation — Talent Protocol operator verification
5. Payments — MPP + x402 agnostic adapter

## KEY MOAT
EP proof_hash embedded INSIDE MPP receipt.
One receipt = payment verified (Stripe/MPP) + authorization verified (EP).
No other agent infrastructure layer has dual verification in a single receipt.
